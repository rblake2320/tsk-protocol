TSK — Tumbler-Style Rotating Segment Keys

A key that expires segment by segment. The structure itself is the secret.

Try Live Demo How It Works
Core Properties

Three layers of key entropy

Each TSK key is a composite of independently-rotating segments, assembled in a secret order known only to the server.

⚙

Segment-Level Rotation

Each key contains static, TOTP (30s window), and HOTP (event-counter) segments. A stolen key decays as rotating segments expire independently.

🔒

Structural Secrecy

The server knows which positions hold which segments. The client knows only segment lengths and order -- never absolute positions within the key string.

🛠

Per-Client Tumbler Map

Every provisioned client gets a unique randomized segment layout. No two clients share a structural map. The layout is never revealed, even to the client.
